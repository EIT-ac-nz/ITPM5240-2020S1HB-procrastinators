Our Product

We over the course of 14 entire days, have developed a new breed of operating system, one based on security and user control, We reveal to you:

ProcrastOS

ProcrastOS is the premier operating system for securty, pen-testing, or any privacy concious user.

Why Choose ProcrastOS?
  • Securtiy

    The security features of ProcrastOS are second to none, all incoming and outcoming data is run through onion layered encryption nodes, creating an enviroment with no single point of failure leading to far safer use, albiet at a somewhat slower pace.

  • Privacy

    Our use of onion technology also prevents data breaches as even we have no knowledge of the data being transfered between systems, but on top of that we have also implemented a built in Virtual Private Network that is available for use.

  • Freedom

    We create no restrictions for you in ProcrastOS, everything is open-source if you can code it you can do it, ProcrastOS is primarily based on debian linux for ease of use, but we have also implemented support for not just debian shell but also unbuntu, windows powershell/bash, and even Mac OSX terminal.

  • Pen-Testing

    While the typical end-user might not be interested in Pen-Testing we thought it a good idea to include 20 of the most common Pen-Testing programs such as: 'Aircrack-ng', 'WPScan', and 'Nmap'.


And as you can see from the diagram below our system architecture uses a 'Protection-Ring' for system privileges, meaning that software and code can only penetrate the higher levels thus protecting the invaluable system functions that exist within the lower levels, and with ProcrastOS any added programs or code that attempts to access the lower levels, must be manually approved by the user, this creates even more protection against malicous software or programs.

Agile Project Management

Throughout the life cycle of this project we have been utilizing agile project management techniques; we here at ProcrastOS have been using the ‘Scrum’ methodology to ensure a cohesive and optimal workflow.

The ‘Scrum’ method is a looping formula that starts with planning and then moves onto the work which is then reviewed, each review adds features or parts of the work that was not finished or needs to be fixed to the backlog to be finished in within the next scrum cycle or “sprint”.

Using this method reduces aspects or improvements that could otherwise be forgotten, and if an employee is lost on what to do next they can simply look at the backlog, this is great for reducing waste time thus improving project workflow.

Agile is all about being flexible (hence the name), this means that we focus on short term goals to optimize deployments that the client can see, and that we focus on the main aspects and features of a project before anything that could be considered additional, meaning that the core aspects of a project will always be implemented by deadline and if there are any additional features we could not finish in time we can cut them from the project easily minimizing wasted work.
